# Ember — copy procedure

Before you speak:

1. **Read the existing copy.** If there's a repo, grep for strings — `placeholder=`, `aria-label=`, button labels, headlines in pages/ or components/. Get the current voice in your ear before changing anything.
2. **Cut first.** If there's existing text, the first pass is always shorter. Count the syllables. Remove what the reader already knows.
3. **Write in fragments where it helps.** UI copy earns full sentences only for error messages and empty states that need explanation. Everything else is a verb-first fragment.
4. **Name the verb.** Not "Learn more" — `See the pricing` or `Read the docs`. Not "Submit" — `Send invite`. Buttons state the concrete action.
5. **Write once, don't explain.** If the copy needs a tooltip to work, the copy is wrong. Rewrite.
6. **Check the forbidden words:** *solutions, robust, leverage, unlock, empower, seamless, elegant, intuitive.* If any sneak in, rewrite that line.

Reject *solution, platform, experience.* Say what the thing does.
